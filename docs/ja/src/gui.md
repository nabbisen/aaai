# GUI ガイド

デスクトップアプリを起動する:

```sh
aaai-gui
```

起動すると Opening 画面が開く。ここで比較する 2 つのフォルダを選び、
メインの 3 ペインワークスペースで監査を進める。ウィンドウタイトルには
現在の定義ファイル名が表示され、未保存の変更がある場合は `●` マーカーが
付く。

---

## 1. Opening 画面

監査はここから始まる。画面は 3 つに分かれる: 上部に必須のフォルダ
選択、中央に折りたたみ式のオプション設定、下部に（保存済み
プロジェクトの有無に応じて）Recent プロジェクト一覧、または初回
オンボーディング。

### 2 つのフォルダを選ぶ

画面上部の 2 枚のカード — **Before**（基準）と **After**（監査対象）。
カードの状態は 3 種類:

- **未選択。** `✗ Not selected` と「フォルダを選ぶ」ボタンを表示。
- **有効。** フォルダを選択すると `✓ /path/to/folder` を表示。
- **無効。** パスが存在しない、またはディレクトリでない場合 `⚠ <理由>` を表示。

フォルダを設定する方法:

1. **ボタンをクリック** — OS ネイティブのフォルダピッカーが開く。
2. **ドラッグ&ドロップ** — ファイルマネージャーからフォルダを
   Opening 画面にドロップ。最初の空きカード（Before→After の順）を
   自動で埋める。フォルダ以外（ファイル等）をドロップすると
   インラインエラーを表示。

### オプション設定

カード下部の折りたたみセクション（「Optional settings」）で
2 つのパスを設定できる。空欄のままにするとデフォルト動作で補完される:

| フィールド | 目的 |
|---|---|
| `audit.yaml` パス | 既存の監査定義。空欄の場合、初回保存時に新規作成。 |
| `.aaaiignore` パス | 除外パスを記述した gitignore 形式のファイル。空欄の場合、`<Before>/.aaaiignore` を自動的に探す。 |

両カードが有効な状態になると **監査開始** ボタンが有効化される。
クリックすると現在のパスをプロファイルとして自動保存し（Recent
Projects に追加される）、バックグラウンドスレッドでフォルダの
比較処理を開始する。大量ファイルの場合も UI はレスポンシブを維持する。

### Recent プロジェクト

過去に監査を実行したことがある場合、Opening 画面の下部に
「Recent projects」一覧が表示される。**最終使用日時の新しい順**で
並び、各行には相対タイムスタンプ（「3 分前」「2 日前」など）が表示される。
各行には 2 つのコントロールがある:

- **Open →** — フォルダカードとオプション設定を復元する。
- **×** — そのエントリを一覧から削除する。

**監査開始** をクリックするたびに、現在のパスが Recent Projects に
自動保存・更新される。手動で「プロファイルを保存」する操作は不要。

### 初回オンボーディング

一度も監査を実行していない場合、Recent 一覧の代わりに**3 ステップ**の
クイックスタートパネルが表示される。初回実行後に消える。

### エラーバナー

監査開始時に問題が発生した場合（フォルダが消えた、audit.yaml が
不正、差分処理の失敗など）、スタートボタンの上部にエラーバナーが
2 行で表示される:

- 赤色の **メッセージ** 行 — 何が起きたかを説明する。
- 灰色の **ヒント** 行 — 次に何をすべきかを説明する。

## 2. Main 画面 — 3 ペイン

監査開始後、ワークスペースは 3 ペインレイアウトに切り替わる:
左にファイルツリー、中央に差分ビュー、右にインスペクター。
各ペインの境界はドラッグで変更できる。

上部にツールバー、その下にフィルタバーと検索バー、下部に
アクションバーが配置される。

### ツールバー

ツールバーは 4 つのボタンとステータスバッジで構成される:

| ボタン | 操作 |
|---|---|
| `□ Open` | Opening 画面に戻る。未保存の変更がある場合は[確認ダイアログ](#ナビゲーションガード)が表示される。 |
| `□ Save` | 監査定義をディスクに保存する。 |
| `▶ Run audit` | バックグラウンドスレッドでフォルダ比較と監査を再実行する。 |
| `↑ Export Report` | ネイティブの保存先ダイアログが開く。`.md` ファイル名 → Markdown、`.json` → JSON（拡張子で自動判別）。 |

保存またはエクスポートに成功すると、ボタン横に
`✓ N 分前に保存` のマークが表示される。30 秒ごとに更新。

ツールバー右側:

- 再実行中（バックグラウンド差分処理中）はステータスバッジに
  **再実行中…** がアンバー色で表示される。
- それ以外は **合格**（緑）または **失敗**（赤）。テキストのみで
  判断できるため色に依存しない。

### フィルタバー

ツールバー直下の 4 ボタンでファイルツリーを絞り込む。
監査結果が読み込まれている場合、各ボタンに件数が括弧書きで表示される:

- **All (N)** — すべてのエントリ
- **Changed only (N)** — Unchanged 以外
- **Pending (N)** — 承認が必要なエントリ（期限切れ承認を含む）
- **Failed & Error (N)** — Failed・Error エントリ

選択中のフィルタには背景色のハイライトが付く。

### 検索バー

パスを対象とした大文字小文字を区別しないインクリメンタル検索。
空欄の場合はフィルタ条件に合うすべてのエントリを表示。

### ファイルツリー（左ペイン）

各エントリはステータスアイコン・名前・注釈で表示される:

| アイコン | ステータス | 備考 |
|---|---|---|
| ✓ | OK | 承認済み・監査合格 |
| ⚠ | Pending | 理由未記入、または承認が期限切れ |
| ✗ | Failed | 監査ルールが一致しない |
| ! | Error | 読み取り・評価に失敗 |
| — | Unchanged / Ignored | 対応不要 |

ディレクトリは `▼` / `▶` で折りたたみ可能。ステータスは
色だけでなく記号でも表現する。

### 差分ビュー（中央ペイン）

ファイルを選択すると差分が左右並べて表示される。3 つのタブで
表示モードを切り替えられる:

| タブ | 表示内容 |
|---|---|
| **左右差分** | 左=Before、右=After。 |
| **統合** | `+` / `-` 行マーカー付きの 1 列表示。 |
| **変更のみ** | 変更のない文脈行を非表示にする。 |

追加行は緑のハイライトと `+` プレフィックス、削除行は赤のハイライトと
`-` プレフィックスで表示される。

バイナリファイルの場合、SHA-256 ハッシュ・ファイルサイズ・
一致・不一致の情報が表示される。

ファイルを選択していない場合、中央ペインにはステータス別の
サマリーカードと要対応のトップ項目を示す**ダッシュボード**が
表示される。

### インスペクター（右ペイン）

変更を許容する理由を記録する場所。ツリーでファイルを選択すると
現在の監査エントリが読み込まれ、編集できる。

ヘッダーにはパス・変更種別・ステータスバッジが表示される。
エントリの **Expires at** 日付が過ぎている場合、ヘッダーに
赤色の **EXPIRED** バッジが表示される — 承認を更新するまで
ファイルツリーでは Pending 扱いになる。

ヘッダー下の編集フィールド:

| フィールド | 必須 | 備考 |
|---|---|---|
| Reason | ✅ | 複数行テキストエリア。空欄では承認不可。 |
| Ticket | — | 外部トラッカーへのリンク（例: `JIRA-123`）。 |
| Approved by | — | レビュアーの名前または ID。 |
| Expires at | — | `YYYY-MM-DD`。この日付を過ぎると自動的に Pending に戻る。 |
| Strategy | — | None / Checksum / LineMatch / Regex / Exact。選択に応じてフォームが変わる。 |
| Note | — | 自由記述メモ。判定に影響しない。 |

承認済みエントリ（OK 状態）の場合、インスペクター下部に
**Pending に戻す** ボタンが表示される。これにより承認が取り消され、
エントリは Pending 状態に戻り、バックグラウンド再実行が起動する。
前回のセッションで承認したエントリを再確認したい場合に有効。
キーボードショートカット `Ctrl+Shift+Z` でも同じ操作ができる。

### ボトムアクションバー

**承認して保存** ボタンは必須フィールドが全て入力済みかつ
ストラテジーが有効な場合のみ有効化される。クリックすると
エントリを承認し、定義を保存し、バックグラウンド再実行を起動する。

バー右側には `N / M 件が未解決` が表示される。

### ナビゲーションガード

未保存の変更がある状態で **Open** をクリックすると
確認ダイアログが表示される:

- **保存して離れる** — 定義を保存してから Opening 画面へ移動する。
  保存に失敗した場合（定義ファイルパス未設定など）はエラーを表示し、
  Main 画面に留まる。
- **破棄して離れる** — 未保存の変更を破棄して Opening 画面へ移動する。
- **キャンセル** — Main 画面に留まる。`Escape` やバックドロップクリックも同様。

## 3. 設定

フッターの **⚙** ボタンで設定ダイアログを開く。設定は
`~/.aaai/prefs.yaml` にセッションをまたいで永続化される。

| 設定 | 説明 |
|---|---|
| **言語** | UI を英語と日本語で切り替える。保存時に即時反映。 |
| **除外ディレクトリ** | プロジェクトごとの `.aaaiignore` よりも先に、すべての監査で除外されるディレクトリ名。デフォルト: `.git`, `target`, `node_modules`, `.DS_Store`。 |

**+ ディレクトリを追加** でディレクトリ名（フルパスではなく
ディレクトリ名のみ、例: `dist`）を追加できる。保存で適用、
キャンセルで破棄。

## 4. キーボードショートカット

Main 画面で **?** キーを押すか、フッターの **?** ボタンを
クリックするとショートカット一覧のオーバーレイが開く。

| ショートカット | 操作 |
|---|---|
| `Ctrl+S` | 定義ファイルを保存 |
| `Ctrl+R` | 監査を再実行 |
| `Ctrl+Z` | 最後の承認を取り消し |
| `Ctrl+Shift+Z` | 選択中の OK エントリを Pending に戻す |
| `Ctrl+E` | レポートを出力（保存先ダイアログが開く） |
| `↑` / `↓` | ファイルツリーの選択を移動 |
| `Tab` / `Shift+Tab` | ペインフォーカスを切り替え |
| `Enter` | インスペクターの Reason フィールドにフォーカス |
| `/` | 検索バーにフォーカス |
| `?` | キーボードショートカット一覧の表示 / 非表示 |
| `Escape` | 開いているオーバーレイを閉じる、またはエントリの選択解除 |

---

## 5. フッター

フッターはウィンドウ下部に表示される:

| 要素 | 意味 |
|---|---|
| `● 未保存の変更` | 承認がディスクにまだ保存されていない場合に表示 |
| `?` | キーボードショートカット一覧を開く |
| `⚙` | 設定ダイアログを開く |
| バージョン | aaai-gui のバージョン |

## 6. レポート

ツールバーの **↑ Export Report** ボタンでネイティブの保存先
ダイアログが開く。ファイル名の拡張子でフォーマットが決まる:

- `.md` — Markdown（人間向け、デフォルトの提案）
- `.json` — JSON（CI パイプライン・ツール向け）

同じ操作は `Ctrl+E` でも行える。

非インタラクティブな環境（CI/CD パイプラインなど）では CLI を使用:

```sh
aaai report --left ./before --right ./after \
            --config audit.yaml --format json --out report.json
```

全オプションは [CLI リファレンス](cli.md) を参照。

---

## 7. 典型的なワークフロー

```
1. aaai-gui を起動する。
2. Before フォルダを画面にドラッグ、After をその上にドロップ、Start をクリック。
3. ダッシュボードで変更の全体像を把握する。
4. ファイルツリーから Pending のエントリを選択する。
5. 中央ペインで差分内容を確認する。
6. インスペクターに理由を記述し、必要に応じてストラテジーを選択する。
7. 「承認して保存」をクリック（または Enter → Ctrl+S）。
8. Ctrl+R で再実行し、ステータスバッジが緑になることを確認する。
9. Ctrl+E でレポートを出力。Markdown または JSON を選択する。
```

## 8. アクセシビリティ（ABDD）

aaai-gui は [ABDD 監査シート](abdd-audit.md) に記録されている
*Accessible by Default Design* 原則に従う:

- ステータスは記号とテキストで表現し、色だけに依存しない
- キーボードですべての操作が完結する
- タップ領域は タッチ操作に十分なサイズ（≥ 44 px）
- エラーメッセージは `メッセージ + ヒント` の 2 行パターン
- 主要操作と破壊的操作を視覚的に区別する

iced 0.14 はウィジェットツリーをプラットフォームの
アクセシビリティ API に公開していないため、スクリーンリーダーの
相互運用性は v1.0 では**未対応**。スクリーンリーダー利用者には
CLI の使用を推奨する。
