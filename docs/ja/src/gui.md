# GUI ガイド

デスクトップアプリを起動する:

```sh
aaai-gui
```

起動すると Opening 画面が開く。ここで比較する 2 つのフォルダを選び、
メインの 3 ペインワークスペースで監査を進める。

---

## 1. Opening 画面

監査はここから始まる。画面は 3 つに分かれる: 上部に必須のフォルダ
選択、中央に折りたたみ式のオプション設定、下部に（保存済みプロジェ
クトの有無に応じて）Recent プロジェクト一覧、または初回オンボー
ディング。

### 2 つのフォルダを選ぶ

画面上部の 2 つのフォルダカード — **Before**（基準）と **After**
（監査対象）。各カードは 3 つの状態を持つ:

- **未選択**: `✗ 未選択` と「フォルダを選ぶ」ボタン。
- **有効**: フォルダを選ぶと `✓ /path/to/folder` 表示。
- **無効**: パスが存在しない、ディレクトリではない等の場合
  `⚠ <理由>` を表示。

カードを満たす方法は 2 つ:

1. **ボタンをクリック** — OS ネイティブのフォルダピッカーが開く。
2. **ドラッグ&ドロップ** — ファイルマネージャからフォルダを画面上の
   どこかにドラッグ。ドラッグ中は上部にヒントバナーが表示される。
   ドロップは「最初に空のカード」（Before → After の順）に格納される。
   **ファイル**（フォルダ以外）をドロップすると、ルールを説明する
   インラインエラーが表示される。

### オプション設定

カードの下に折りたたみ式の「オプション設定」セクション。空欄なら
デフォルト動作で推論される 2 つのパスを露出している:

| フィールド | 用途 |
|---|---|
| `audit.yaml` のパス | 既存の監査定義。空欄なら空の定義から開始。 |
| `.aaaiignore` のパス | gitignore 形式の除外ルール。空欄なら `<Before>/.aaaiignore` を自動探索。 |

両方のフォルダカードが有効になると **監査を開始** ボタンが有効化
される。クリックするとバックグラウンドスレッドでフォルダ比較を実行
（大規模ツリーでも UI は応答を維持する）。

### Recent プロジェクト

過去に監査したプロジェクトが 1 件以上保存済みなら、Opening 画面の
下部に「最近使ったプロジェクト」一覧が表示される。並び順は
**最終使用、新しいものから**。各行に短い相対時刻（例: "3 min ago"、
"2 d ago"、1 週間以上前なら ISO 形式の絶対日付）を表示。
*Open* をクリックするとフォルダカードと保存済みオプション設定が
復元され、そのプロジェクトのタイムスタンプも更新される。

### 初回オンボーディング

プロジェクトをまだ一度も保存していない場合、Recent 一覧の代わりに
**はじめての方へ** パネルが表示される。3 ステップ（Before を選ぶ、
After を選ぶ、開始をクリック）を列挙し、`audit.yaml` が初回保存時
に自動生成される旨も添える。一度プロジェクトを保存するとこの
パネルは消える。

### エラーバナー

監査開始時に何か問題が起きた場合（ピック後にフォルダが消えた、
audit.yaml が壊れている、フォルダ比較が失敗した等）、開始ボタンの
上にエラーバナーが 2 行で表示される:

- **メッセージ**行: 赤色で「何が起きたか」。
- **ヒント**行: グレーで「次に何をするか」。

これはアプリ全体で「行動可能なエラー」に使われる共通パターン
（message + hint）。

## 2. メイン画面 — 3 ペイン

開始すると 3 ペインレイアウトに切り替わる: 左にファイルツリー、
中央に差分ビューア、右にインスペクター。ペイン間のディバイダは
ドラッグ可能で、画面サイズに合わせて比率を調整できる。

上部には toolbar、その下にフィルターバーと検索バー、下部には
アクションバーが横一文字に並ぶ。

### ツールバー

ツールバーには 4 つのボタンとステータスバッジ:

| ボタン | 動作 |
|---|---|
| `□ 開く` | Opening 画面に戻る。未保存変更があれば警告。 |
| `□ 保存` | 監査定義をディスクに保存。 |
| `▶ 監査実行` | 現在の差分と定義に対して監査を再実行。 |
| `↑ レポート出力` | Markdown レポートを `aaai-report.md` に出力。 |

保存またはエクスポートが成功すると、該当ボタンの隣に小さい緑の
チェックマーク + 相対時刻が表示される（例: `✓ 保存 2 min ago`）。
このラベルは 30 秒ごとに更新される（少なくとも 1 回保存または
エクスポートが実行された場合のみ）。次回の監査再実行でクリア
される。

ツールバー右側には **監査ステータス** バッジ: 直近の監査結果に
応じて **Passed**（緑）または **Failed**（赤）を表示。テキスト
で表記するため、グレースケール表示でも判定が見える。

### フィルターバー

ツールバー下に 4 つのボタンでファイルツリーを絞り込む:

- **All** — すべて表示
- **Changed only** — Unchanged を非表示
- **Pending** — 理由と承認がまだ必要なエントリ
- **Errors** — Failed と Error のエントリ

現在のフィルタは薄い背景色でハイライトされる。

### 検索バー

フィルターバーの右の検索欄は、エントリパスに対する大文字小文字
無視のインクリメンタル部分一致検索。空欄ならフィルタが表示する
全件を表示。

### ファイルツリー（左ペイン）

各エントリは 1 ファイルまたは 1 ディレクトリ。ステータスアイ
コン + 名前 + 必要なら注釈で表示:

| アイコン | ステータス | 補足 |
|---|---|---|
| ✓ | OK | 理由あり、監査通過 |
| ⚠ | Pending | 理由未入力、レビュー待ち |
| ✗ | Failed | 監査ルールが一致しない |
| ! | Error | 読み取り・評価不能 |
| — | Unchanged / Ignored | 操作不要 |

ディレクトリは `▼` / `▶` の三角で折りたたみ可能。ステータス
アイコンは色だけでなく記号で識別できる（ABDD アクセシビリティ
方針の一環）。

監査を実行していない状態（Opening と Start の間など）では、
ファイルツリーに小さなプレースホルダーパネルが表示され、
ツールバーの `▶ 監査実行` ボタンへ誘導する。

### 差分ビューア（中央ペイン）

ファイルを選択すると、そのコンテンツが左右並列で表示される。
差分上部の 3 タブで表示モードを切り替えられる:

| タブ | 表示内容 |
|---|---|
| **左右差分** | 左 = Before、右 = After。両側に行番号付き。 |
| **統合** | 単一列に `+` / `-` 行マーカーを表示。 |
| **変更のみ** | 変化のない context 行を非表示。 |

追加行は緑系の背景、削除行は赤系の背景で表示。各行の先頭に
`+` / `-` 文字が付くため、色だけに依存せず識別可能。

バイナリファイル（行単位比較が適さない）の場合、SHA-256 ハッシュ・
ファイルサイズ・内容一致の有無を専用パネルで表示。

監査結果がロードされておらず、ファイルも選択されていない場合、
中央ペインには代わりに **ダッシュボード** が表示される: ステータス
別のサマリカード（OK / Pending / Failed / Error / Ignored）と、
注意が必要な上位エントリ一覧。

### インスペクター（右ペイン）

インスペクターは「なぜその変更を許可するか」を記録する場所。
ファイルツリーで選択するとそのエントリの現在の AuditEntry が
ロードされる。保存するとメモリ上の定義に反映され、ツールバーの
保存ボタンをクリックでディスクに永続化される。

ヘッダーにはエントリのパス、変更種別（Added / Removed /
Modified / TypeChanged）、現在のステータスバッジを表示。

ヘッダーの下に編集可能フィールド:

| フィールド | 必須 | 補足 |
|---|---|---|
| 理由 | ✅ | 監査の中心。複数行のテキストエリア。空欄 = 承認不可。 |
| チケット | — | 外部トラッカーへのリンク（例: `JIRA-123`） |
| 承認者 | — | レビュアーの名前または ID |
| 有効期限 | — | この承認が失効する日付 `YYYY-MM-DD`（任意） |
| 戦略 | — | None / Checksum / LineMatch / Regex / Exact。下のフィールドは戦略に応じて変化。 |
| メモ | — | 自由記述。判定には影響しない。 |

戦略ごとに異なる入力フォームが現れる。例えば Regex はパターン
フィールドと共にリアルタイムバリデーションを表示。パターンが
コンパイル不能の場合、フィールド直下に regex101.com への
リンクを含むヒント付きインラインエラーが出る。

何も選択されていない場合、インスペクターには「ファイルを選んで
ください」とファイルツリーへ誘導する小さなプレースホルダー
パネルが表示される。

### ボトムアクションバー

ワークスペースの下部には主要操作ボタン **承認して保存** が
1 つ。インスペクターのすべての必須フィールドが埋まり、戦略が
有効な場合のみ enable される。クリックすると現在のエントリを
承認し、定義を保存し、監査を再実行してステータスバッジを
更新する。

## 3. キーボードショートカット

| ショートカット | 動作 |
|---|---|
| `Ctrl+S` | 監査定義ファイルを保存 |
| `Ctrl+R` | 監査を再実行 |
| `Ctrl+Z` | 最後の承認を取り消し |
| `Ctrl+E` | Markdown レポートをエクスポート |
| `↑` / `↓` | ファイルツリーで選択を移動 |
| `Enter` | インスペクターの理由欄にフォーカス |
| `Escape` | 現在の選択を解除 |

これらは v1.x で安定 — 詳細は [互換性方針](compatibility.md)
を参照。

---

## 4. フッター

フッターは画面下部に横一文字に並ぶ:

| 要素 | 意味 |
|---|---|
| `● 未保存の変更があります` | ディスクへ未保存の承認が 1 件以上あるとき表示 |
| ショートカット凡例 | キーボードショートカットの簡易表示（メイン画面のみ） |
| 言語ピッカー | 日本語 / English を切り替え |
| バージョン番号 | aaai-gui のバージョン |

---

## 5. レポート出力

ツールバーの **↑ レポート出力** ボタンは Markdown レポートを
カレントディレクトリの `aaai-report.md` に出力する。同じ操作は
`Ctrl+E` にも割り当て済み。

JSON 出力や CI パイプラインへの組み込みには CLI を使う:

```sh
aaai report --format json --out aaai-report.json
```

レポートオプションの全体像は [CLI リファレンス](cli.md) を参照。

## 6. 典型的なワークフロー

```
1. aaai-gui を起動
2. Before フォルダを画面にドラッグ、After を続けてドラッグ、開始ボタンをクリック
3. ダッシュボードで全体の変更状況を把握
4. ファイルツリーで Pending エントリを選択
5. 中央ペインで差分を確認
6. インスペクターで理由を入力、必要に応じて戦略を選択
7. 承認して保存 をクリック（または Enter → Ctrl+S）
8. Ctrl+R で再実行してステータスバッジが緑になることを確認
9. Ctrl+E で Markdown レポートを出力
```

保存ボタンの隣の小さな `✓ 保存 Nm ago` で「最後にいつ保存
したか」が分かる。エクスポートも同様。ツールバー右端の監査
ステータスバッジは、現在の判定に同期して表示される。

---

## 7. アクセシビリティ (ABDD)

aaai-gui は [ABDD 監査シート](abdd-audit.md) に記録されている
*Accessible by Default Design* の原則に従う:

- ステータスは色だけでなく記号 + テキストで伝える
- すべての操作にキーボードで到達可能、フォーカスが視認できる
- クリック領域は ≥ 44 px（タッチ操作対応）
- エラーメッセージは「message + hint」の 2 行パターン
- 主要操作と破壊的操作は視覚的に分離

**スクリーンリーダー対応は v1.0 では実装されていない**。
GUI ツールキット（iced 0.14）がプラットフォームのアクセシビリティ
API にウィジェットツリーを公開していないため。スクリーンリーダー
を必要とするユーザーには、iced がこの機能を実装するまで CLI の
利用を推奨する。
