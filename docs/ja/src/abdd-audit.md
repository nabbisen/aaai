# ABDD 監査シート — vX.Y.Z

> **目的.** リリースごとに、GUI が設計書 `aaai_uiux_design.pdf` p.8 の
> Accessible by Default Design (ABDD) チェック観点を満たしているかを記録する
> ファイル。GUI を実機で動かす担当者が「実観測」と「判定」列を埋める。
> RFC 017 の視覚検証カードを補完し、a11y のみに特化した記録となる。
>
> **書き方.**
>   1. GUI をビルド: `cargo build -p aaai-gui --release`。
>   2. 各セクションの「Where」列で示された画面に遷移して期待挙動を確認。
>   3. **実観測** (1 文) と **判定** を以下のいずれかで埋める:
>      - `✅ 合格`
>      - `❌ 不合格` — 追跡用に Issue または RFC 番号を **備考** に記録
>      - `例外` — v1.0 出荷はする。理由を **備考** に記録
>   4. 検証完了後、タイトルの `vX.Y.Z` を実バージョンに置換。
>
> **スコープ.** v1.0.0 が ABDD として保証するのは
> 「キーボードで完結する」「色だけに依存しない」「フォーカスが常に見える」まで。
> スクリーンリーダー対応は §7 に記載の理由で v1.x 以降に延期。

## 2. 色なしでの判別可能性 (設計書 p.8)

モノクロ / グレースケール表示で実行する (Linux: `xrandr --output <name> --gamma 1:1:1` / macOS: 設定 → アクセシビリティ → ディスプレイ → カラーフィルタ → グレースケール)。

| 何を | 期待 | 実観測 | 判定 | 備考 |
|---|---|---|---|---|
| ファイルツリーのステータスアイコン | `✓ ⚠ ✗ ! —` が相互に判別可能 | | | |
| ツールバーの監査ステータス | "Passed" / "Failed" のテキストが判定要素となり、色だけに依存しない | | | |
| 差分ビューアの追加/削除行 | 行頭の `+` / `−` 文字が残っており、それで識別できる | | | |
| インスペクター LineMatch ルールブロック | 行頭 `+` / `−` および `action:` ラベルが維持されている | | | |

## 3. 主操作と破壊的操作の距離 (設計書 p.8)

| ペア | 期待 | 実観測 | 判定 | 備考 |
|---|---|---|---|---|
| 保存して次へ vs ナビゲーションガードの破棄 | 分離済み（RFC 086:「その他の選択肢」に隠れている）; 視覚的重みが異なる | | | |
| 保存して次へ vs ルール削除 | 別リボン / 隣接しない | | | |
| 保存 vs 再実行 | ツールバー上で隣接するが、ラベル・アイコンが明確に区別される | | | |

## 4. 未保存・監査失敗・入力エラー状態が常に見える (設計書 p.8)

| 状態 | 表示位置 | 実観測 | 判定 | 備考 |
|---|---|---|---|---|
| 未保存変更 | ツールバー / ウィンドウタイトル / 保存ボタン状態 | | | |
| 監査結果 (Passed / Failed) | ツールバー右側、常時表示 | | | |
| インスペクターの入力エラー (例: 不正な正規表現) | フィールドの直下、message + hint 2 行 | | | |
| Opening のフォルダ未検出エラー | 監査開始ボタンの上のバナー、message + hint 2 行 | | | |

## 7. スクリーンリーダー対応 — v1.0 制限事項

iced 0.14 は OS のアクセシビリティ API (Linux の AT-SPI、Windows の UIA、
macOS の NSAccessibility) にウィジェットツリーを公開しない。iced がこの
フックを提供するまで、スクリーンリーダー利用者にとって aaai-gui は実用的
ではない。

**v1.0.0 はこれを既知の制限事項として宣言する**。スクリーンリーダーが必要な
ユーザーには `aaai` CLI を推奨経路として案内する。

iced が a11y サポートを実装した時点で再評価し、本セクションを埋めるか、
新規 RFC で移行作業を立ち上げる。
